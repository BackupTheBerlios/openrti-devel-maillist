On Tuesday, December 17, 2013 09:53:14 Eric Noulard wrote:
> I would suggest to find another way to seek for relative path.
> Pretty much like $ORIGIN does for RPATH.
I went ahead and imported what I did for the python patch into this current 
project.
On systems that implement a way to find the path of a specific binary, this 
includes all already supported win32 targets as well as the usual known linux 
variants at least, you should now be able to move the whole installed tree to 
somewhere else and still find the management object model.

What is not yet solved is the case where you build in your build tree but do 
not install. This is the place where you probably want to run the test suite 
without installing it. Not sure what is the best solution for this.
