> 
> - join federation
> - enable time regulation
> - enable time constrained
> - publish & subscribe
> - wait for synchronization
> - register instance
> - wait for second synch point
> - run
> 
> If I instantiate only one federate then here is no problem.
> 
> If I instantiate more than one federate (same code) then at the point

I think you run on a well know problem with a federate willing to get time 
regulating needs an information that is currently handed over by other 
federates in internal message processing. That is if the other federates are 
in one of the evokeCallback variants the problem does not show up.

I have an idea to change this, but did not find the time to implement that idea 
yet. May be this works out these days.
