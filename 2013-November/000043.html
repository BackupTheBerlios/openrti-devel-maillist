Hello,

I'm using OpenRTI from release-0.4 branch.
I found a bug in BasicDataElements encoding functions.
Some encoded values are not decoded correctly and produce wrong results.
This bug occurs for large numbers for all integer and floating point types that are more than 1 Byte long.
std::string and std::wtring are also affected by the bug.

Please find in attachment one patch that adds a unit test for encoding/decoding of basic data elements ;
and one patch that fix my problem.

I only tested this fix on VisualStudio 2012 32bits.
