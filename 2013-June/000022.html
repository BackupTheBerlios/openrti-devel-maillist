I assume you are on windows since you mention cmake-gui.
Start cmake-gui, enter a build directory and the source directory.
Select a generator, press configure and generate. Then use visual c to compile 
the code.

> Also I need to know if I can compile with gcc mingwin.
Currently mingw is untested and due to the cyclic link dependency of the rti 
abi most probably does not compile. Neither does cygwin. Visual c should work, 
at least I regularily compile and test this here on a virtual machine.
What is missing is some adjustments to the build system to do the same kind of 
tricks OpenRTI already does with visual c.

At this moment I try to build a next windows test setup with mingw installed. 
So, that should not be too much work, but without a test setup here, I guess 
we will not have too much luck ...

Appart from that, it would be interresting to know how you interface with the 
rti api. The choice of the compiler together with matlab can lead to more or 
less problems if you try to integrate something.
