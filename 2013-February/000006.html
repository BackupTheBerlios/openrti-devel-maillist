> The self tests care for their own server infrastructure.
> They set up a tree of server nodes where I wanted to have the federates at 
> defined leafs of that tree. Therefore all the federates already get their 
> address and protocol assigned according to the -A and -S argument to the 
> tests. If you give additional ones on the command line you will end up having 
> duplicate argument keys for the connect call. I am currently not sure if the 
> first or the last one of identical keys wins - well I would call this 'behaves 
> undefined'.
> 
> If you want to get a 'connection refused' you can try fddget just beside the 
> other tests which resolves and prints handles.
> 
> Try
> 
> fddget -O <object model> -o <object class name> -a <attribute name>
> 
> If you now add the 'protocol=rti address=localhost' as the last argument I get 
> an error message if rtinode is not running on localhost.
