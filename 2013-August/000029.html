Hello,

I have a link error in Visual Studio when compiling an example project containing a class that inherit from rti1516e::NullFederateAmbassador.

The easiest way to fix this problem is to create a file NullFederateAmbassador.cpp in your rti1516e project containing only these lines :
// This time, the first include is above the api include.
// the rti1516/Exception header misses that.
#include <iosfwd>
#include <RTI/NullFederateAmbassador.h>

You can see a good  explanation of the problem in Portico RTI source code :
https://github.com/calytrix/portico/blob/master/codebase/src/cpp/ieee1516e/src/fedamb/NullFederateAmbassador.cpp
